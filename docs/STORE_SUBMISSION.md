# Store submission checklist

Use this when publishing **Background Reveal** to Firefox Add-ons (AMO), Chrome Web Store, and Microsoft Edge Add-ons.

## Before you submit (repo / manifest)

These are **your** decisions or one-time setup — not auto-generated by the build.

| Item | Status in repo | Action required |
|------|----------------|-----------------|
| **Extension ID (Firefox)** | `background-reveal@local.dev` in [`wxt.config.ts`](../wxt.config.ts) | Replace with a permanent ID you control, e.g. `reveal@yourdomain.com`, before AMO listing. AMO assigns/adds IDs on first upload in some flows, but a stable `gecko.id` is required for updates. |
| **Version** | `0.1.0` in `wxt.config.ts` / `package.json` | Bump for each store release. |
| **Privacy policy URL** | [`PRIVACY.md`](../PRIVACY.md) in repo only | **Host publicly** (GitHub Pages, repo raw link, or your site). All three stores require a URL in the listing — a file in git is not enough. |
| **Support / contact URL** | Issue link in PRIVACY only | Add a support email or GitHub issues URL in store listings. |
| **Developer accounts** | — | [Firefox AMO](https://addons.mozilla.org/developers/) (free), [Chrome Web Store](https://chrome.google.com/webstore/devconsole) ($5 one-time), [Edge Add-ons](https://partner.microsoft.com/dashboard/microsoftedge/public/login) (free). |
| **Screenshots** | Described in [`store-assets/README.md`](../store-assets/README.md) | Capture manually; add PNG files under `store-assets/screenshots/` when ready. |
| **Promotional copy** | Draft in `store-assets/README.md` | Paste into each store’s listing form; adjust length limits per store. |

## Build artifacts

```bash
npm ci
npm test
npm run build:firefox
npm run build:chrome
npm run lint:ext
npm run zip:firefox    # → .output/*-firefox*.zip (or similar)
npm run zip:chrome     # → .output/*-chrome*.zip
```

**Firefox signed XPI (recommended for AMO):**

```bash
# Create API credentials at https://addons.mozilla.org/developers/addon/api/key/
export AMO_JWT_ISSUER=...
export AMO_JWT_SECRET=...
npm run sign:firefox   # → web-ext-artifacts/*.xpi
```

Upload the signed `.xpi` to AMO, or submit the zip for manual review depending on your AMO workflow.

**Chrome / Edge:** Upload the zip from `npm run zip:chrome`. Edge accepts the same MV3 package in most cases.

## Permissions — reviewer justification

Copy/adapt this in store “notes for reviewer” / permission justification fields:

> **Background Reveal** helps users find image URLs that sites expose via CSS backgrounds, overlays, lazy attributes, or tiled (IIIF) viewers — not plain `<img>` tags.
>
> - **`host_permissions` / `<all_urls>`** — Required only when the user invokes the extension (context menu or picker). The content script reads computed styles, DOM attributes, and may fetch IIIF `info.json` for tiled viewers. No background scanning; no data sent to the extension author.
> - **`contextMenus`** — “Reveal images here” on right-click.
> - **`downloads`** — User-initiated download of a revealed URL.
> - **`clipboardWrite`** — User-initiated copy URL.
> - **`scripting`** — Fallback clipboard helper when page blocks clipboard API.
> - **`activeTab`** — Standard pattern for user-gesture-driven tab access.

## Store-specific requirements

### Firefox Add-ons (AMO)

| Field | Guidance |
|-------|----------|
| **Add-on name** | Background Reveal |
| **Summary** | ≤ 250 chars; see store-assets |
| **Description** | Full feature list + limitations; link to support |
| **Privacy policy** | Public URL to hosted `PRIVACY.md` |
| **License** | MIT (matches repo) |
| **Categories** | Photos, Accessibility, or Developer Tools |
| **Notes to reviewer** | Permission block above + steps: load catalog.archives.gov record, wait for viewer, right-click → Reveal images here |
| **Data collection** | Manifest declares `data_collection_permissions.required: ["none"]` |
| **Source code** | AMO may require source upload for minified builds; this project builds from readable TypeScript — be prepared to link the repo or upload source zip |

### Chrome Web Store

| Field | Guidance |
|-------|----------|
| **Single purpose** | Reveal hidden image URLs on pages the user selects |
| **Privacy practices** | No data collected; no remote code; all processing local |
| **Permission justifications** | One paragraph per sensitive permission (host access, downloads, clipboard) |
| **Privacy policy URL** | Required |
| **Screenshots** | At least 1; 1280×800 or 640×400 recommended |
| **Icon** | 128×128 PNG (from `public/icons/icon-128.png`) |
| **Distribution** | Public or unlisted |

### Microsoft Edge Add-ons

| Field | Guidance |
|-------|----------|
| **Package** | Same MV3 zip as Chrome |
| **Privacy policy URL** | Required |
| **Store listing** | Similar to Chrome; often reuse Chrome copy |
| **Certification** | Automated + manual review; expect `<all_urls>` scrutiny |

## Documents **outside** the repo (you create)

1. **Hosted privacy policy** — same text as `PRIVACY.md`, reachable via HTTPS.
2. **Store listing screenshots** — 3–5 PNGs (see store-assets).
3. **Support channel** — email, GitHub Issues, or support page URL.
4. **AMO API credentials** — JWT issuer/secret for `web-ext sign` (store securely; never commit).
5. **Optional: landing page** — short explanation + install links after approval.

## Pre-submission smoke test

1. Load unpacked build (Firefox + Chrome).
2. [catalog.archives.gov](https://catalog.archives.gov/id/45641547) — viewer loaded → reveal → **iiif full** → Open.
3. CSS background test page or site with hero background — reveal → Open / Download.
4. Multiple reveals without closing panel — list grows; **Open all** / **Download all** work.
5. Close panel (×) — next reveal starts fresh list.

## After approval

- Tag release in git: `v0.1.0`
- Attach store zips or signed XPI to GitHub Release (optional)
- Bump version in `wxt.config.ts` for next development cycle
